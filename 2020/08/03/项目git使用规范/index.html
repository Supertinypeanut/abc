项目git使用规范

项目 git 使用规范

分支规范

功能分支

feature-(姓名简称)-(jira 号/任务 ID)

feature-jzh-XXXXXXXID

上线分支

release-(年月日)

release-20200803

冲突分支

conflict-(姓名简称)-(年月日)

conflict-jzh-20200803

conflict-jzh-20200803-1

测试/灰度修复分支

bugfix-(jira 号/任务 ID)

线上修复分支

hotfix-(年月日)

hotfix-20200803

Commit Message 规范

在 commit 的时候需要增加上面中的类型：

Merge Request 规范

标题格式为：

Code Review

由项目 Leader/同事 进行 review 代码，review 完成之后 进行评论 LGFM 后合并分支
